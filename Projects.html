Search Engine

Description: Under construction. Me and a team of four other University of Michigan students are building a search engine from the ground up! This is a class project from 398 System Design of a Search Engine taught by Nicole Hamilton who worked on the creation of Microsoft's Bing.

Details

There are seven parts to a search engine. Crawler, HTML parser, Inverted index, Ranker, Query Compiler, Constraint Solver and Front end. For my part I implemented an ad-hoc html parser that could extract text in title and anchor tags, as well as regular body text. I ignored script tags because we were not allowed to use annyone else's libraries and making a javascript interpreter was more work than needed. The basic idea was to have a "global" file pointer that began at the beginning of the file and traverse through it, finding a and title tags while ignoring style and script tags.
I also implemented the query compiler, which interpreted different queries using a simple top-down recursive method.

Log Manager

Description: In professional software, rather than outputting messages to the standard output stream (stdout via cout) for the purpose of debugging or indicating errors, developers will typically use log files to hide such information from end-users. These log files have the advantage of being archivable for later examination should an anomaly occur in the software system. Log files are usually quite massive in size and there are often many log entries that may be irrelevant to what the developer is trying to examine. The developer needs to have the ability to quickly search for and analyze only the entries they care about. My program helps with this task by being an interactive interface where the user can perform timestamp, category, and keyword searches for the purpose of constructing an "excerpt list" of the log file. My program also allows the user to manage and display this "excerpt list" to identify the important/relevant entries of their log file.

Details

I gained experience writing code that makes use of multiple interacting data structures.

Image Processing

Description: A seam carving alogirthm. Seam carving is a technique for content-aware resizing of images (sometimes known as "retargeting"). The end result is that we can resize images in a way that changes the aspect ratio but does not distort the image. The algorithm works by finding and removing "seams" in the image that pass through the least important pixels. My program only implements shrinking the dimensions of an image. For more info on seam carving, watch this youtube video. Or read up on it here.


TSP Zookeeper

As a zookeeper in a zoo with a bunch of animals, everyday the zookeeper should provide food and water for the animals. In order to minimize the expenses of the zoo, it's necessary to devise a systematic way to feed the animals. The plan is to first make paths to each animal cage to deliver food, and then to construct canals to provide fresh water to all animal cages. My program identifies the most efficient routes for providing​​ food​​ and​​ water​​ to​​ the​​ animals. It has two modes. The first, is a simple MST to make paths to the cages to feed the animals minimizing the expenses of building paths by finding the most efficient routes. The second mode, is a more true TSP problem where I find the most cost efficient routes for constructing water canals to the cages. In order to have water flow within the canals, the​​ route​​ should​​ be​​ a ​​cycle. Calculates the MST cost for 40,000 cages and TSP cost for 40 cages.

Details

Implemented an MST using Prim's algorithm instead of Kruskal's because the graph was sparse and contained many more edges than vertices making its Big O faster. Learned about branch and bound method and applying heuristics.
